複合選擇器介紹

一、類別選擇器 vs 複合選擇器​

  1. 類別選擇器、偽類選擇器

    部分名稱說明
    .card類別選擇器（Class Selector）選擇 class 為 card 的元素
    :nth-child(2)結構性偽類選擇器（Structural pseudo-class）選擇「第 2 個子元素」

  2. 複合選擇器（Compound Selector）

    • .card:nth-child(2) 由以下組成
      • 一個類別選擇器（.card）
      • 一個偽類選擇器（:nth-child()）

  3. 程式範例

    <div class="container">
      <div class="card">1</div>
      <!-- child 1 -->
      <div class="card">2</div>
      <!-- child 2 -->
      <div class="card">3</div>
      <!-- child 3 -->
      <div class="card">4</div>
      <!-- child 4 -->
    </div>
    • .card:nth-child(2) ：選中父元素的第 2 個子元素，而這個子元素的 class 必須是 .card。
      • ✅ 適用：.card 剛好是第 2 個子元素時
      • ❌ 不適用：若 .card 是第 3、4... 個子元素，就不會被選中
    • .card:nth-child(2n) ：選中第 2、4、6… 個子元素中，class 為 .card 的元素。
      • ✅ 適用：選中所有落在偶數順序的 .card
      • ❌ 不適用： 如果 .card 沒落在偶數位置，就不會被選中
    • .card:nth-of-type(2)：選中第二個 .card 類型的元素（例如第二個 <div> 且 class 是 .card）
      • ✅ 適用：不管中間有沒有其他類別，只看相同 HTML 標籤類型
      • 常用於 .card 是某一種特定標籤（如 <div>）
    • .card:nth-of-type(2n)：選中所有 .card 元素中，順序為第 2、4、6… 的那幾個（根據 HTML tag 類型）
      • ✅ 適用： 選中所有偶數順序的 .card
      • 比 :nth-child() 更穩定（因為不受其他不同類型標籤干擾）
