響應式導覽列轉換（直立 ⇄ 橫向滑動）

一、前言​

在 RWD 設計中，當分類選單項目較多時：

  • 桌機版（大螢幕）：寬度夠，適合放在側邊欄，採用直立排列（Vertical）。
  • 手機版（小螢幕）：直立會佔用過多垂直空間（導致使用者一直往下滾動）。最佳解法是改為頂部橫向排列（Horizontal），並允許水平溢流滾動（Scroll），同時隱藏滾動條以求美觀。

二、技術拆解​

  • 三個關鍵技術點：
  1. 防止 Flex 擠壓變形 (flex-shrink: 0)
    • 在手機版橫排時，我們通常會使用 display: flex。但 Flexbox 預設會拼命把所有子元素壓縮，好塞進同一個畫面中（這會導致選單文字斷行、變形）。
    • 解法：在子項目（.nav-item）加上 flex-shrink: 0。
    • 效果：強制子項目「不准縮小」，維持自己原本設定好的舒適寬度（Padding 撐開的寬度）。
  2. 水平溢流與阻尼感 (overflow-x & touch)
    • 在手機版，我們必須限制外容器不能超出螢幕，但內容器可以自由往右延伸。
    • 解法：
      • 外層容器設定 width: 100% 加上 overflow-x: auto。
      • 加上 white-space: nowrap，強制裡面的文字絕對不換行。
      • 加上 webkit-overflow-scrolling: touch，讓 iOS 的 Safari 或 Chrome 瀏覽器在用手指滑動選單時，擁有像原生 App 一樣流暢、有彈性的「阻尼感（回彈效果）」。
  3. 利用 Media Query 進行「排版大洗牌」
    • 我們以 Bootstrap 5 的 lg 斷點（992px）為分界點：
      • < 992px（手機版）：flex-direction: row（橫排），外層開啟 overflow-x: auto。
      • ≥ 992px（桌機版）：將 flex-direction 轉為 column（直排），外層關閉滾動（overflow-x: visible）、允許正常換行（white-space: normal）。

三、名詞解釋​

  1. overflow 系列：當子元素大於父元素時，決定如何處理「溢出」的部分。

    屬性作用
    overflow-x管理水平(左右)溢出
    overflow-y管理垂直(上下)溢出
    auto(值)內容溢出時才顯示滾動條,乾淨不佔空間
    hidden(值)直接切掉溢出部分,看不見也摸不到

  2. white-space: nowrap：強制不換行

    • **用途：**水平滑動選單的靈魂屬性
    • **不加的後果：**選單項目會掉到第二行，滾動效果做不出來

  3. w**ebkit-overflow-scrolling: touch**：流暢度補強

    • **效果：**手指放開後產生「慣性回彈」的阻尼感
    • **不加的後果：**滑動感生硬，像在滑一塊木頭

  4. 消失的滾動條

    • **目的：**可以滾動，但不顯示醜醜的拉桿。
寫法對應瀏覽器
&::-webkit-scrollbar { display: none; }Chrome、Safari、Edge
-ms-overflow-style: none;IE、舊版 Edge
scrollbar-width: none;Firefox

💡

三段必須一起寫,才能跨瀏覽器全部隱藏。

  1. align-items: stretch：垂直拉皮
  • **預設值：**這是 Flexbox 的預設值，常被忽略
  • **實戰意義：**直立選單中，文字長短不一時,背景顏色仍會橫向撐滿整齊

  1. Flex 三兄弟：這三個屬性是 Flex 子元素的核心,決定它們如何「分配空間」。

    1. flex-grow

      • **解釋：**當父容器有「多餘空間」時，我要改變大小嗎?
      • 預設值:0：保持原本大小
      • **設為 1 以上：**把多餘空間吃掉
      • **多個元素都設值時：**按比例分配
        • 例如 A=1、B=2，B 會搶到 A 的兩倍空間

    2. flex-shrink

      • **解釋：**當父容器「空間不夠」時，我願不願意提壓縮自己的空間
      • 預設值：1：願意被壓縮
      • 設為 0：不願意被壓縮
      • **重要性：**水平滾動選單中,，設 0 會導致分類文字被擠到變形或疊在一起

    3. flex-basis

      • **解釋：**在使用 grow / shrink 之前，我「原本」應該有多寬(或多高)？
      • 預設值：auto 根據內容自動決定
      • **可設定值：**像 200px、50%、10rem 等
      • 與 width 的差別:
        • width 是「死的」尺寸
        • flex-basis 是「起始」尺寸，之後可被 grow/shrink 調整
        • 同時設定時，flex-basis 會勝出

    4. 對照表

      屬性預設值控制情境設為 0 的意義
      flex-grow0空間有剩保持原本大小
      flex-shrink1空間不夠不願意被壓縮
      flex-basisauto起始尺寸從 0 開始計算

    5. 縮寫語法 flex

      /* 完整寫法 */
      flex-grow: 1;
      flex-shrink: 1;
      flex-basis: 0;

      /* 縮寫(順序固定) */
      flex: 1 1 0;

      /* 常見快捷值 */
      flex: 1; /* 等於 1 1 0%,最常用,平均分配空間 */
      flex: auto; /* 等於 1 1 auto,依內容大小彈性分配 */
      flex: none; /* 等於 0 0 auto,完全不彈性,寫死尺寸 */

  2. transition ：讓 CSS 從「瞬間切換」變成「平滑動畫」。

    • 完整語法結構 → 四個參數依序排列，順序固定不能亂寫。 
      transition: [屬性] [持續時間] [速度曲線] [延遲時間];
    • 四個參數詳解
      1. 屬性(transition-property)
        • **作用：**指定要監控「哪個屬性」的變化
        • 常用值：
          • all：監控所有屬性變化(最方便但效能較差)
          • 指定單一屬性：background-color、transform、opacity
          • 多個屬性：用逗號分隔 background-color, transform
      2. 持續時間(transition-duration)
        • 作用:動畫要花多久時間完成
        • 常用值:
          • 0.2s:符合人體工學的「快感」(按鈕、Hover 最佳值)
          • 0.3s:稍微慢一點,適合卡片、選單
          • 0.5s 以上:大型元件、頁面切換
        • 單位:s(秒)或 ms(毫秒,1s = 1000ms)
      3. 速度曲線(transition-timing-function)
        • **作用：**動畫的「節奏感」,先快還是先慢
        • 常用值：
          值節奏使用場景
          linear從頭到尾等速進度條、跑馬燈
          ease慢→快→慢(預設值)一般通用
          ease-in慢慢起步,後面變快元素離開畫面
          ease-out快速起步,慢慢停下元素進入畫面
          ease-in-out慢→快→慢(更明顯)質感最好,通用
          cubic-bezier(...)自訂曲線客製化動畫
      4. 延遲時間(transition-delay)
        • **作用：**動畫「等多久」才開始
        • 預設值：0s(立即執行)
        • 使用場景：做連鎖動畫、tooltip 提示框延遲顯示
    • 常見錯誤與注意事項
      1. transition 寫在「原本的狀態」，不是 :hover
    /* ❌ 錯誤 */
    .btn:hover {
      background: red;
      transition: all 0.2s; /* 寫這裡只有 hover 進去會動畫,離開瞬間切換 */
    }

    /* ✅ 正確 */
    .btn {
      transition: all 0.2s; /* 寫在原始狀態,進出都有動畫 */
    }
    .btn:hover {
      background: red;
    }

  3. 多行文字截斷：當文字太長時,只顯示 N 行,多餘的用「...」表示。

    • 多行截斷(顯示 3 行) 
      .text-clamp {
        display: -webkit-box;
        -webkit-box-orient: vertical;
        -webkit-line-clamp: 3;
        overflow: hidden;
        text-overflow: ellipsis;
      }
    • 單行截斷 
      .single-line {
        white-space: nowrap; /* 呼應第 2 點 */
        overflow: hidden; /* 呼應第 1 點 */
        text-overflow: ellipsis;
      }
    • **注意：**不能與 display: flex 同時使用，如需 flex 排版請在內層多包一層元素。
    • 屬性介紹：
      屬性作用
      display: -webkit-box啟用可限制行數的特殊盒子
      -webkit-box-orient: vertical文字垂直排列
      -webkit-line-clamp: N限制顯示 N 行
      overflow: hidden藏起多餘內容
      text-overflow: ellipsis截斷處顯示「...」
